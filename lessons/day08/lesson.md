# Day 08: Vimマクロ（q, @） × Bashのgrep・正規表現

## 今日のゴール（15分で達成）

- Vimマクロで「同じ編集を何行にも一瞬で適用」できる
- `grep` の基本オプションと正規表現でログやコードを絞り込める
- マクロで生成したコードを `grep` でフィルタリングして確認できる

---

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|----------|------|
| `qa` | レジスタ `a` にマクロ記録開始 |
| `q` | マクロ記録終了 |
| `@a` | レジスタ `a` のマクロを1回実行 |
| `5@a` | マクロを5回繰り返す |
| `@@` | 直前に実行したマクロを再実行 |

### マクロの基本手順

```
1. qa      ← レジスタaに記録開始（左下に "recording @a" と表示）
2. ...操作  ← 繰り返したい編集を1回行う
3. q       ← 記録終了
4. @a      ← 再生（何度でも）
5. 10@a    ← 10回繰り返す
```

> **コツ**: マクロ内で `j` を使って「次の行へ移動」を入れると、`10@a` で10行まとめて処理できる。

## Bash チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|----------|------|
| `grep "pattern" file` | ファイルからパターンを検索 |
| `grep -n "pattern" file` | 行番号付きで表示 |
| `grep -i "pattern" file` | 大文字小文字を無視 |
| `grep -v "pattern" file` | マッチしない行だけ表示 |
| `grep -E "pat1\|pat2" file` | 拡張正規表現（OR検索） |

### よく使う正規表現

```bash
grep "^ERROR" log.txt        # ERRORで始まる行
grep "\.sh$" filelist.txt    # .shで終わる行
grep "[0-9]\{3\}" data.txt   # 数字3桁を含む行
grep -E "foo|bar" file.txt   # fooまたはbarを含む行
grep -v "^#" script.sh       # コメント行を除外
```

---

## 組み合わせポイント（なぜ一緒に学ぶか）

マクロは「同じ編集の繰り返し」を自動化するVimの機能。  
grepは「パターンに一致する行の抽出」をするBashの機能。

**組み合わせると**:  
マクロで大量のコードを生成 → grep で意図通りに出力されているか確認  
という「生成と検証」のサイクルを素早く回せる。
