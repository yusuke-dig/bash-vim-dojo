# Day 14: :g コマンド × xargs

## 今日のゴール（15分で達成）

- Vimの `:g` コマンドでパターンにマッチする行を一括操作できる
- `xargs` でコマンドの出力を別コマンドの引数として渡せる
- `:g` で特定行を抽出 → ファイル保存 → `xargs` で処理する流れを体験する

---

## Vim チートシート（今日覚える3〜5コマンド）

| コマンド | 動作 |
|---------|------|
| `:g/パターン/コマンド` | パターンにマッチする全行にコマンドを実行 |
| `:g/パターン/d` | パターンにマッチする全行を削除 |
| `:g/パターン/p` | パターンにマッチする全行を表示 |
| `:v/パターン/d` | パターンにマッチ**しない**全行を削除（`:g!` と同じ） |
| `:g/パターン/s/old/new/g` | パターン行の中だけ置換する |

### 使用例

```
:g/^#/d          → コメント行（# で始まる行）をすべて削除
:g/error/p       → "error" を含む行をすべて表示
:v/function/d    → "function" を含まない行をすべて削除
:g/TODO/s/TODO/DONE/g  → TODO を含む行の TODO を DONE に置換
```

## Bash チートシート（今日覚える3〜5コマンド）

| コマンド | 動作 |
|---------|------|
| `コマンド1 \| xargs コマンド2` | コマンド1の出力をコマンド2の引数として渡す |
| `xargs -I {} コマンド {}` | `{}` を引数の置換文字として使う |
| `xargs -n 1 コマンド` | 引数を1つずつ渡して繰り返し実行 |
| `find . -name "*.txt" \| xargs grep "keyword"` | 複数ファイルをまとめてgrep |
| `echo "file1 file2" \| xargs rm` | 空白区切りのリストをコマンドに渡す |

### 使用例

```bash
# ファイル名のリストを引数に渡す
echo "a.txt b.txt" | xargs ls -l

# -I {} で引数の位置を明示
ls *.sh | xargs -I {} echo "Found: {}"

# find と組み合わせて一括操作
find . -name "*.log" | xargs rm

# -n 1 で1件ずつ処理
echo "apple banana cherry" | xargs -n 1 echo "Item:"
```

## 組み合わせポイント（なぜ一緒に学ぶか）

`:g` は「Vim内でパターン行を処理する」コマンド、`xargs` は「コマンド出力をシェルで繰り返し処理する」ツール。

どちらも「**複数の対象をまとめて操作する**」という発想が共通している。

- Vim内でファイルを編集 → パターン行を `:g` で抽出・整形
- ファイル名リストを `find` で生成 → `xargs` で一括処理

「テキスト加工はVim、ファイル操作はシェル」という使い分けが身につく。
