## Vim チートシート（今日覚える3〜5コマンド）

| コマンド | 動作 |
|---------|------|
| `=G` | カーソル行からファイル末尾までインデントを自動整形 |
| `gg=G` | ファイル全体のインデントを整形 |
| `>>`/`<<` | 現在行を1段インデント増加 / 減少 |
| `]p` | インデントを合わせてペースト |
| `:set list` | タブ・スペースを可視化（インデントのズレ発見に便利） |

> YAML はインデントが命。`gg=G` や `:set list` でズレを防ごう。

## Bash チートシート（今日覚える3〜5コマンド）

| 書き方 | 意味 |
|--------|------|
| `cmd > file` | 標準出力をファイルへ書き込み（上書き） |
| `cmd >> file` | 標準出力をファイルへ追記 |
| `cmd 2> file` | 標準エラーをファイルへ書き込み |
| `cmd 2>/dev/null` | エラーを捨てる（表示しない） |
| `cmd > file 2>&1` | 標準出力・エラーを両方同じファイルへ |
| `cmd &> file` | 上と同じ（bash 4 以降の短縮形） |

### コード例（そのまま使える）

```bash
# 標準出力だけファイルへ
ls -la > output.txt

# エラーメッセージを捨てる
find / -name "*.conf" 2>/dev/null

# 出力もエラーも全部ログへ
./deploy.sh > deploy.log 2>&1

# 追記モード
echo "step done" >> build.log

# 標準出力はファイル、エラーは別ファイル
make build > build.log 2> error.log
```

## 組み合わせポイント（なぜ一緒に学ぶか）

CI スクリプト（GitHub Actions の `run:` ブロックなど）では、

1. **YAML** でワークフローを定義し
2. **シェルスクリプト** 内でリダイレクトを使ってログを整理する

この 2 つをセットで扱えると、実際の CI 設定を自分で読み書きできる。  
Vim で YAML を素早く編集し、スクリプト部分のリダイレクトも一発で直せるのが今日のゴール。
