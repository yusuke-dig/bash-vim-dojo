# Day 05: Vimヤンク・ペースト（y, p, yy） × Bash関数

## 今日のゴール（15分で達成）

- Vimの `yy`・`p` でコードブロックをコピー＆ペーストできる
- Bashで引数・ローカル変数を持つ関数を定義・呼び出しできる
- 既存の関数を `yy/p` でコピーして新しい関数のベースとして再利用できる

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|----------|------|
| `yy` | カレント行をヤンク（コピー） |
| `p` | カーソルの次の行にペースト |
| `P` | カーソルの前の行にペースト |
| `yw` | カーソルから単語末尾までヤンク |
| `3yy` | 現在行から3行ヤンク |

> **ヤンク（yank）** = Vimでの「コピー」。クリップボードではなくVimの内部レジスタに入る。

## Bash チートシート（今日覚える5コマンド）

| 書き方 | 動作 |
|--------|------|
| `greet() { ... }` | 関数定義（`function` キーワードは省略可） |
| `greet` | 引数なし関数の呼び出し |
| `$1`, `$2` | 関数の第1・第2引数 |
| `local var=value` | ローカル変数（関数内のみ有効） |
| `result=$(func arg)` | 関数の echo 出力を変数に受け取る |

## 組み合わせポイント（なぜ一緒に学ぶか）

Bash関数は構造が似た形の繰り返しになることが多い。
`yy` で既存の関数ブロックをコピー → `p` で貼り付け → `:s` で名前を書き換えると、
関数をゼロから書くより断然速い。Day 04 の `:s` 置換と組み合わせると効果倍増！

```
# 元の関数を3行ヤンク（例: 関数が3行の場合）
3yy

# 下に貼り付け
p

# 関数名を置換（カーソルを貼り付けた関数名の行に移動してから）
:s/greet/farewell/g
```
