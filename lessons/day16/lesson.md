## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|---------|------|
| `vimdiff file1 file2` | 2ファイルを差分表示で開く |
| `]c` | 次の差分箇所へジャンプ |
| `[c` | 前の差分箇所へジャンプ |
| `do` | diff obtain — 相手ウィンドウの内容を自分側に取り込む |
| `dp` | diff put — 自分の内容を相手ウィンドウに送る |
| `:qa` | 全ウィンドウを閉じる |

### vimdiff の使い方

```bash
# ターミナルで実行
vimdiff buggy.sh correct.sh
```

```
┌──────────────────┬──────────────────┐
│  buggy.sh        │  correct.sh      │
│                  │                  │
│  echo $n * 2  ◄──┤──echo $((n*2))  │  ← 差分がハイライト
│                  │                  │
└──────────────────┴──────────────────┘
```

- `]c` で次の差分へ移動
- 左ウィンドウにいるとき `do` で right → left にコピー
- 確認したら `:qa` で終了

## Bash チートシート（今日覚える4コマンド）

| コマンド | 動作 |
|---------|------|
| `bash -n script.sh` | シンタックスチェック（実行せずに構文確認） |
| `bash -x script.sh` | トレース実行（各コマンドを stderr に表示） |
| `set -x` / `set +x` | スクリプト内でトレースを ON / OFF |
| `set -e` | エラー発生時に即終了 |
| `set -u` | 未定義変数をエラーとして扱う |

### よく使うパターン

```bash
# シンタックスチェック
bash -n script.sh && echo "SYNTAX OK"

# トレース実行（stderr に詳細が出る）
bash -x script.sh

# スクリプト内でトレース区間を絞る
set -x
result=$((a + b))   # この行だけトレース
set +x

# 安全なスクリプトの先頭定型文
set -e
set -u
```

## 組み合わせポイント（なぜ一緒に学ぶか）

`vimdiff` はVimのウィンドウ分割機能の応用。Day 7 で学んだ `:sp / :vsp` の発展版。

```
デバッグの流れ:
  bash -n script.sh   → シンタックスエラーを発見
  bash -x script.sh   → 実行トレースで論理エラーを発見
  vimdiff buggy correct → 正解と比較して修正箇所を特定
  Vim で編集 → :w で保存 → bash script.sh で確認
```
