# Day 02: Vim挿入・削除（i, a, x, dd） × Bashのif文・比較演算子

## 今日のゴール（15分で達成）

- Vimで文字・行を挿入・削除できる
- Bashのif文で条件分岐を書ける
- 条件分岐スクリプトをVimで編集して動作確認できる

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|---------|------|
| `i` | カーソルの**前**に挿入（Insert mode開始） |
| `a` | カーソルの**後ろ**に挿入（Append） |
| `A` | 行末に挿入 |
| `x` | カーソル上の1文字を削除 |
| `dd` | 現在行を丸ごと削除 |

**挿入モードの抜け方**: `Esc` キー（必ず戻ること！）

### よくある操作パターン
```
# 単語の末尾に追記したい → a
# 行末に追記したい    → A
# 文字を1つ消したい  → x
# 行ごと消したい     → dd
# 消した行を戻したい → u（undo）
```

## Bash チートシート（今日覚える5構文）

### if文の基本構造
```bash
if [ 条件 ]; then
  # 条件が真のとき
elif [ 別の条件 ]; then
  # 別の条件が真のとき
else
  # どれも偽のとき
fi
```

### 数値の比較演算子

| 演算子 | 意味 | 例 |
|--------|------|-----|
| `-eq` | 等しい (equal) | `[ $a -eq 10 ]` |
| `-ne` | 等しくない (not equal) | `[ $a -ne 10 ]` |
| `-lt` | より小さい (less than) | `[ $a -lt 10 ]` |
| `-le` | 以下 (less or equal) | `[ $a -le 10 ]` |
| `-gt` | より大きい (greater than) | `[ $a -gt 10 ]` |
| `-ge` | 以上 (greater or equal) | `[ $a -ge 10 ]` |

### 文字列の比較

| 演算子 | 意味 |
|--------|------|
| `=` | 文字列が等しい |
| `!=` | 文字列が等しくない |
| `-z` | 文字列が空 |
| `-n` | 文字列が空でない |

```bash
name="Alice"
if [ "$name" = "Alice" ]; then
  echo "こんにちは、Alice！"
fi
```

### 注意点
- `[` と `]` の内側にスペースが必要：`[ $a -eq 1 ]` ✅ / `[$a -eq 1]` ❌
- 変数はダブルクォートで囲む：`"$var"` ✅（スペースを含む値で壊れない）

## 組み合わせポイント（なぜ一緒に学ぶか）

if文のスクリプトを書くとき、よくあるのが「条件の値を変えたい」「分岐を追加したい」という編集。
- 数値を変えるなら `x` で1文字消して `i` で書き直す
- elif節を追加するなら `A` で行末に移動して改行・入力
- 不要な分岐を削除するなら `dd` で丸ごと消す

VimとBashのif文は「素早く正確に編集する」という同じ目的でつながっている。
