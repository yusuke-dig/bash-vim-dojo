# Day 15: 折りたたみ（za, zc, zo） × 長いスクリプトの構造化

## 今日のゴール（15分で達成）

- Vimの折りたたみコマンドで長いファイルをセクションごとに折りたたんで全体把握できる
- `# {{{` / `# }}}` マーカーを使ってBashスクリプトをセクション分けできる
- 関数・main パターンを使って長いスクリプトを読みやすく構造化できる

## Vim チートシート（今日覚える3〜5コマンド）

| コマンド | 動作 |
|---------|------|
| `za` | カーソル位置の折りたたみを開閉（トグル） |
| `zc` | カーソル位置の折りたたみを閉じる |
| `zo` | カーソル位置の折りたたみを開く |
| `zM` | ファイル内のすべての折りたたみを閉じる |
| `zR` | ファイル内のすべての折りたたみを開く |

### 折りたたみを使うための設定

```vim
:set foldmethod=marker
```

このコマンドを実行すると、`{{{` を含む行で折りたたみが始まり、`}}}` を含む行で終わる。

### Bashスクリプトでの折りたたみマーカー

```bash
# {{{ Section 1: 初期化
init() {
    echo "初期化処理"
}
# }}}

# {{{ Section 2: メイン処理
main() {
    init
    echo "メイン処理"
}
# }}}
```

Vimで開いて `:set foldmethod=marker` を実行すると、各セクションを折りたたんで全体構造が一目でわかる。

## Bash チートシート（今日覚える3〜5コマンド）

| パターン | 動作 |
|---------|------|
| `関数名() { ... }` | 関数を定義する |
| `local 変数名=値` | 関数内のローカル変数 |
| `echo "[$level] $msg"` | 変数を埋め込んで出力 |
| `if [ -f "$path" ]; then ... fi` | ファイル存在チェック |
| `main() { ... }; main` | スクリプトを main 関数で包むパターン |

### 使用例

```bash
# セクション分けの例（{{{ と }}} が折りたたみマーカー）

# {{{ ログ出力関数
log_message() {
    local level=$1
    local msg=$2
    echo "[$level] $msg"
}
# }}}

# {{{ ファイルチェック関数
check_file() {
    local path=$1
    if [ -f "$path" ]; then
        echo "EXISTS: $path"
    else
        echo "MISSING: $path"
    fi
}
# }}}

# {{{ メイン処理
main() {
    log_message "INFO" "処理開始"
    check_file "/tmp/test.txt"
}
main
# }}}
```

## 組み合わせポイント（なぜ一緒に学ぶか）

スクリプトが長くなると「全体像が見えない」問題が起きる。

- Bashの「セクション分け＋関数化」でコードを論理的に整理する
- Vimの「折りたたみ」で物理的に不要なセクションを隠す

`# {{{` / `# }}}` マーカーはBashのコメントなので実行時は無視される。VimとBash両方にとって意味のある書き方ができる。
