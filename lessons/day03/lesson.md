# Day 03: Vim検索（/, n, N, *） × Bashのfor文・seq

## 今日のゴール（15分で達成）

- Vimで `/` を使ってキーワードを検索し、`n` / `N` で移動できる
- `*` で同じ単語をすばやく検索できる
- Bashの `for` 文と `seq` でループを書ける
- ループスクリプトの中を Vim 検索で素早くナビゲートできる

## Vim チートシート（今日覚える4コマンド）

| コマンド | 動作 |
|---------|------|
| `/パターン` + Enter | 下方向に検索（カーソルが一致箇所へジャンプ） |
| `n` | 次のマッチへ移動（Next） |
| `N` | 前のマッチへ移動（Next逆方向） |
| `*` | カーソル下の単語を検索（次のマッチへ） |

### 補足
- 検索はファイル末尾を超えると先頭に折り返す（ラップアラウンド）
- ハイライトを消したいときは `:noh` + Enter
- 大文字小文字を区別しない検索は `/パターン\c`

### よくある操作パターン
```
# ループ変数 "i" が出てくる場所をすべて確認したい → /\bi\b Enter → n n n
# 関数名 "greet" にカーソルを置いて全出現を探したい → *
# 誤って進みすぎた → N で一つ戻る
```

## Bash チートシート（今日覚える3構文）

### for文の基本構造
```bash
for 変数 in リスト; do
  # ループ内の処理
done
```

### リストをその場で書く
```bash
for fruit in apple banana cherry; do
  echo "$fruit"
done
```

### seq でカウントアップ
```bash
# seq 開始 終了
for i in $(seq 1 5); do
  echo "回数: $i"
done
```

### seq でステップ指定
```bash
# seq 開始 ステップ 終了
for i in $(seq 0 2 10); do
  echo "$i"   # 0 2 4 6 8 10
done
```

### 範囲をブレース展開で書く（seqの代替）
```bash
for i in {1..5}; do
  echo "$i"
done
```

## 組み合わせポイント（なぜ一緒に学ぶか）

for文のスクリプトを書くと、ループ変数や `done` が何度も登場する。
長いスクリプトで「このループどこで終わってる？」「変数 `i` を使っている行は全部で何行？」を
調べるとき、`/` 検索と `n` ジャンプは**ファイル内 grep** のように働く。

- ループ変数名を `*` で検索 → 使われている全箇所を一瞬で把握
- `for` や `done` を `/` 検索 → ネストの深さを目で追える
- 行頭の `#` コメントを `/^#` で検索 → 演習のタスク番号を一気に探せる

## 振り返り問題

演習が終わったら、以下の問いに頭の中で答えてみる。詰まったらチートシートを見直す。

**Q1.** `/apple` で検索後、3つ先のマッチへ移動するにはキーを何回押す？

<details><summary>答え</summary>
`n` を3回。
</details>

**Q2.** カーソルが `fruit` という単語の上にある。`/fruit` を手打ちせずに同じ検索をかける方法は？

<details><summary>答え</summary>
`*` を押す。カーソル下の単語が自動的に検索パターンになり、次のマッチへジャンプする。
</details>

**Q3.** `seq 1 5` と `{1..5}` はどちらも同じ数列を生成するが、`{1..5}` が使えない場面はどんなとき？

<details><summary>答え</summary>
ステップや終端値を変数で指定したいとき。`{1..$n}` は展開されない（Bashのブレース展開は変数を評価しない）ので、`$(seq 1 $n)` を使う必要がある。
</details>

**Q4.** 以下のコードは何を出力する？頭の中で実行してみる。

```bash
for i in $(seq 2 2 6); do
  echo "x$i"
done
```

<details><summary>答え</summary>

```
x2
x4
x6
```

`seq 2 2 6` は「2から始まり2ずつ増やして6まで」→ `2 4 6`。
</details>

**Q5.** 検索ハイライトが残って邪魔なとき、消すコマンドは？

<details><summary>答え</summary>
`:noh` + Enter。「no highlight」の略。
</details>
