# Day 03: Vim検索（/, n, N, *） × Bashのfor文・seq

## 今日のゴール（15分で達成）

- Vimで `/` を使ってキーワードを検索し、`n` / `N` で移動できる
- `*` で同じ単語をすばやく検索できる
- Bashの `for` 文と `seq` でループを書ける
- ループスクリプトの中を Vim 検索で素早くナビゲートできる

## Vim チートシート（今日覚える4コマンド）

| コマンド | 動作 |
|---------|------|
| `/パターン` + Enter | 下方向に検索（カーソルが一致箇所へジャンプ） |
| `n` | 次のマッチへ移動（Next） |
| `N` | 前のマッチへ移動（Next逆方向） |
| `*` | カーソル下の単語を検索（次のマッチへ） |

### 補足
- 検索はファイル末尾を超えると先頭に折り返す（ラップアラウンド）
- ハイライトを消したいときは `:noh` + Enter
- 大文字小文字を区別しない検索は `/パターン\c`

### よくある操作パターン
```
# ループ変数 "i" が出てくる場所をすべて確認したい → /\bi\b Enter → n n n
# 関数名 "greet" にカーソルを置いて全出現を探したい → *
# 誤って進みすぎた → N で一つ戻る
```

## Bash チートシート（今日覚える3構文）

### for文の基本構造
```bash
for 変数 in リスト; do
  # ループ内の処理
done
```

### リストをその場で書く
```bash
for fruit in apple banana cherry; do
  echo "$fruit"
done
```

### seq でカウントアップ
```bash
# seq 開始 終了
for i in $(seq 1 5); do
  echo "回数: $i"
done
```

### seq でステップ指定
```bash
# seq 開始 ステップ 終了
for i in $(seq 0 2 10); do
  echo "$i"   # 0 2 4 6 8 10
done
```

### 範囲をブレース展開で書く（seqの代替）
```bash
for i in {1..5}; do
  echo "$i"
done
```

## 組み合わせポイント（なぜ一緒に学ぶか）

for文のスクリプトを書くと、ループ変数や `done` が何度も登場する。
長いスクリプトで「このループどこで終わってる？」「変数 `i` を使っている行は全部で何行？」を
調べるとき、`/` 検索と `n` ジャンプは**ファイル内 grep** のように働く。

- ループ変数名を `*` で検索 → 使われている全箇所を一瞬で把握
- `for` や `done` を `/` 検索 → ネストの深さを目で追える
- 行頭の `#` コメントを `/^#` で検索 → 演習のタスク番号を一気に探せる
