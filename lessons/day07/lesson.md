# Day 07: Vimウィンドウ分割（:sp, :vsp） × Bashパイプ・リダイレクト

## 今日のゴール（15分で達成）

- Vimでウィンドウを縦・横に分割してファイルを並べて見られる
- Bashのパイプ（`|`）とリダイレクト（`>`, `>>`, `<`）を使ってデータを流せる
- 分割画面でスクリプトを書きながら、その出力をリダイレクトで確認できる

---

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|----------|------|
| `:sp ファイル名` | 横分割（上下）でファイルを開く |
| `:vsp ファイル名` | 縦分割（左右）でファイルを開く |
| `Ctrl-w w` | 次のウィンドウへ移動（順番に） |
| `Ctrl-w h/j/k/l` | 左/下/上/右のウィンドウへ移動 |
| `:q` | 現在のウィンドウを閉じる（他ウィンドウは残る） |

### 補足
- `:sp` なしで `:vsp` だけだと同じファイルを2画面で見ることができる（左右で見比べ）
- `Ctrl-w =` で全ウィンドウを均等サイズに揃えられる

---

## Bash チートシート（今日覚える5コマンド）

| 構文 | 動作 |
|------|------|
| `cmd1 \| cmd2` | cmd1の出力をcmd2の入力に渡す（パイプ） |
| `cmd > file` | cmdの出力をfileに書き込む（上書き） |
| `cmd >> file` | cmdの出力をfileに追記する |
| `cmd < file` | fileの内容をcmdの入力に渡す |
| `cmd 2> file` | エラー出力をfileに書き込む |

### よく使う組み合わせ
```bash
ls -la | grep ".sh"          # ls の出力を grep に渡す
echo "hello" > out.txt       # ファイルに書き出し
echo "world" >> out.txt      # ファイルに追記
cat out.txt | wc -l          # 行数を数える
sort < list.txt > sorted.txt # listを並び替えてsortedに保存
```

## 組み合わせポイント（なぜ一緒に学ぶか）

**「書く」と「確認」を同時に行う**

スクリプトを開発するとき、よくある流れは：
1. スクリプトを書く（Vim）
2. 実行して出力をファイルに保存（リダイレクト）
3. 出力ファイルを確認する（別ターミナル or 別ウィンドウ）

Vimのウィンドウ分割を使えば、`:vsp output.txt` で出力ファイルを右側に表示しながら、左でスクリプトを編集できる。パイプを使えば中間ファイルなしにコマンドをつなげて確認もできる。
