# Day 13: タブ（:tabnew, gt） × awk基礎

## 今日のゴール（15分で達成）

- Vim のタブ機能でスクリプトとデータファイルを別タブで開いて編集できる
- `awk` でフィールドの抽出・条件フィルタ・集計ができる
- タブでデータを見ながら awk スクリプトを書くワークフローを体験する

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|---------|------|
| `:tabnew ファイル名` | 新しいタブでファイルを開く |
| `gt` | 次のタブへ移動 |
| `gT` | 前のタブへ移動 |
| `:tabc` | 現在のタブを閉じる |
| `:tabs` | タブ一覧を表示 |

### タブの使い方（典型パターン）

```
# ターミナルで
vim exercise.sh

# Vim の中で
:tabnew data.txt    ← data.txt を別タブで開く
gt                  ← exercise.sh タブへ戻る
gT                  ← data.txt タブへ戻る
:tabc               ← 現在のタブを閉じる
```

## Bash チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|---------|------|
| `awk '{print $1}' file` | 各行の1番目のフィールドを出力 |
| `awk '$2 > 70' file` | 2番目のフィールドが70より大きい行を出力 |
| `awk '{sum += $2} END {print sum}' file` | 2番目のフィールドの合計を出力 |
| `awk '{print $1": "$2}' file` | フィールドを組み合わせて出力 |
| `awk 'NR==1' file` | 1行目だけ出力（NR = 行番号） |

### awk の基本構造

```bash
awk 'パターン { アクション }' ファイル

# パターンなし: 全行に適用
awk '{print $1}' data.txt

# 条件パターン
awk '$2 > 70 {print $1}' data.txt

# BEGIN / END ブロック
awk 'BEGIN{print "開始"} {print $1} END{print "終了"}' data.txt
```

### awk の特殊変数

| 変数 | 意味 |
|------|------|
| `$0` | 行全体 |
| `$1`, `$2`, ... | 各フィールド（スペース区切り） |
| `NR` | 現在の行番号 |
| `NF` | 現在行のフィールド数 |

## 組み合わせポイント（なぜ一緒に学ぶか）

データ分析の典型ワークフロー:

```
Tab 1: data.txt  ← データの中身を確認しながら
Tab 2: exercise.sh ← awk コマンドを書く
```

`data.txt` を見ながら `exercise.sh` を編集できるため、フィールド番号のミスが激減する。
`gt` / `gT` でタブを切り替えてデータ確認 → コード修正を繰り返す。
