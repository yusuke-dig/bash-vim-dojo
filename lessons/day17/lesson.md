# Day 17: ctags / タグジャンプ × 関数の整理・モジュール化

## 今日のゴール（15分で達成）

- `ctags` でタグファイルを生成し、`Ctrl-]` で関数定義へジャンプできる
- Bash 関数を別ファイル（lib.sh）に分けて `source` で読み込める
- Vim と ctags を組み合わせて、複数ファイルを素早く移動できる

## Vim チートシート（今日覚える5コマンド）

| コマンド | 動作 |
|---------|------|
| `:! ctags -R .` | カレントディレクトリ以下のタグファイルを生成 |
| `Ctrl-]` | カーソル下の識別子の定義へジャンプ |
| `Ctrl-t` | ジャンプ前の場所に戻る |
| `:tag 関数名` | 指定した関数の定義へジャンプ |
| `:ts` | 同名タグの候補一覧を表示（tselect） |

## Bash チートシート（今日覚える5コマンド）

| 書き方 | 動作 |
|--------|------|
| `function 名前() { ... }` | 関数を定義する |
| `$1`, `$2` | 関数の引数を受け取る |
| `echo "値"` | 関数の戻り値として出力する |
| `source lib.sh` | 別ファイルの関数を読み込む |
| `result=$(関数名 引数)` | 関数の出力を変数に格納する |

### コード例（そのまま写せば動く）

```bash
# 関数定義
function greet() {
    echo "Hello, $1!"
}

# 関数呼び出し
greet "World"      # → Hello, World!

# 別ファイルに分けた関数を読み込む
source lib.sh

# 関数の出力を変数に格納
result=$(greet "Vim")
echo "$result"     # → Hello, Vim!
```

```bash
# add 関数：2つの引数の合計を返す
function add() {
    echo $(( $1 + $2 ))
}
add 3 5    # → 8
```

```bash
# to_upper 関数（lib.sh に書く例）
function to_upper() {
    echo "$1" | tr '[:lower:]' '[:upper:]'
}
to_upper "bash"    # → BASH
```

## 組み合わせポイント（なぜ一緒に学ぶか）

実務のスクリプトは関数が複数ファイルに分かれることが多い。`ctags` があれば：

- `greet "World"` の `greet` の上で `Ctrl-]` → 定義へ即ジャンプ
- `Ctrl-t` で呼び出し元に戻る
- `:tag to_upper` で lib.sh の関数定義に直接飛ぶ

ファイルをまたいでも迷わず移動できるのが強み。

## 演習の進め方

1. `vim exercise.sh` で開く
2. `:! ctags -R .` でタグファイルを生成（`tags` ファイルが作られる）
3. Task 1〜2: exercise.sh に greet と add 関数を定義する
4. `:e lib.sh` で lib.sh を開き、to_upper 関数を定義する（`:wq` で保存）
5. `greet` や `to_upper` の上で `Ctrl-]` → 定義へジャンプしてみる
6. `Ctrl-t` で戻る、`:tag 関数名` でも試す
7. `:e exercise.sh` で戻り Task 3〜5 を完成させる（`:wq` で保存）
8. `bash test.sh` で採点する

演習が終わったら以下を試してみよう：

- `result=$(add 10 20)` のように変数に格納して `echo "$result"` で表示する
